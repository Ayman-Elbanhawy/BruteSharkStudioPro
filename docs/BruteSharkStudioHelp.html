BruteShark Desktop Studio v2.3.0

BruteShark Desktop Studio is a Windows network forensic analysis application for reviewing packet captures, reconstructing sessions, extracting credentials and hashes, detecting suspicious traffic, fingerprinting TLS/SSH activity, carving files, and exporting professional evidence reports.

Primary usersIncident responders, security analysts, network engineers, lab researchers.
InputsPCAP, PCAPNG, CAP, and live network capture streams.
OutputsTables, carved artifacts, Hashcat files, HTML reports, and project files.
Installer bundleDesktop app, help manual, README, screenshot, and Hashcat v7.1.2.

Install and Requirements

Install the MSI from the repository release folder:

release\BruteSharkDesktopStudioInstaller.msi

The installer creates desktop and Start menu shortcuts and installs Hashcat under the application folder.

Recommended environment

  • Windows 10 or later.
  • .NET 8 Desktop Runtime, if not already available on the workstation.
  • Npcap or a compatible capture driver for live capture.
  • Administrator rights for installer execution and some live capture interfaces.
  • Enough disk space for carved files, exported reports, and large PCAP processing.

Quick Start

  1. Launch BruteShark Desktop Studio.
  2. Click + Add and select one or more capture files.
  3. Keep TCP Sessions: ON and UDP Sessions: ON enabled for deeper parsing.
  4. Confirm the desired modules are checked in the Modules panel.
  5. Click Run or Analyze.
  6. Use the left navigation tree to inspect credentials, hashes, sessions, files, detections, fingerprints, protocols, and statistics.
  7. Click Export to write evidence artifacts and a full HTML report.
  8. Click Save to preserve the investigation as a .bsproj file.
Good practice: Start with a small capture to validate module selection, then process larger collections once the workflow is confirmed.

Offline Capture Analysis

Offline analysis is the safest and most repeatable workflow for forensic review.

  1. Add capture files through + Add or the large plus button.
  2. Remove unwanted captures with the minus button.
  3. Use module checkboxes to limit analysis scope when working with very large captures.
  4. Click Run to process all loaded captures.
  5. Watch the progress bar and file status column for processing state.

Session reconstruction options

Leave TCP and UDP session reconstruction enabled when you need file carving, credential parsing, VoIP extraction, and application-layer protocol analysis. Disable a session type only when you need faster high-level processing and understand the loss of deeper parsing.

Live Capture

  1. Select a network interface in the dropdown.
  2. Optionally enter a BPF filter such as tcp port 443, udp, or host 192.168.1.10.
  3. Enable Promiscuous Mode only when the investigation requires it.
  4. Click Start.
  5. Click Stop before exporting a stable result set.
Capture driver note: Live capture depends on a compatible capture driver such as Npcap and may require elevated privileges.

Save and Open Projects

Use project files when you need to pause, hand off, or preserve a case.

  • Save or Ctrl+S writes a .bsproj project file.
  • Open or Ctrl+O restores a saved project.
  • Clear resets the current workspace after confirmation of workflow intent.

Hashcat Workflow

Hashcat v7.1.2 is bundled with the installer.

  1. Analyze a capture that contains authentication material.
  2. Open Credentials > Hashes.
  3. Select the hash type.
  4. Choose an output folder and a wordlist.
  5. Optionally add extra Hashcat arguments.
  6. Click Create Hashcat file or Crack with Hashcat.

Troubleshooting

The app cannot capture live traffic

  • Install or repair Npcap.
  • Run the application as administrator.
  • Confirm a network interface is selected.
  • Remove or simplify the BPF filter and try again.

Results look incomplete

  • Confirm TCP/UDP session reconstruction is enabled.
  • Verify relevant modules are checked before running analysis.
  • Use a complete capture rather than a truncated packet sample.

Hashcat does not start

  • Confirm Hashcat exists under the installed Hashcat folder.
  • Restart the terminal or workstation if PATH was updated during installation.
  • Provide a valid wordlist path.

The help button cannot open this manual

The MSI installs this file under docs\BruteSharkStudioHelp.html next to the application folder. If the file is missing, repair the installation or reinstall the MSI.

Responsible Use

Use BruteShark Desktop Studio only on captures and networks you are authorized to inspect. Packet captures can contain credentials, personal data, and regulated information. Store exports and project files according to your organization's evidence handling and data retention policies.
